# Geospatial Analytics and Demand Modeling for Data-Driven Retail Expansion

## Project Overview

This project focuses on preparing a retail dataset for geospatial analytics and demand modeling. The objective is to identify data quality issues, clean and transform the data, and create an analysis-ready dataset that can support strategic retail expansion decisions.

## Objectives

* Perform data profiling and quality assessment
* Identify and handle missing values
* Detect and remove duplicate records
* Standardize and transform data fields
* Perform feature engineering
* Generate a clean dataset for further analytics

## Tools Used

* Python
* Pandas
* VS Code
* Microsoft Excel
* GitHub

## Dataset Details

Initial Dataset:

* 200 records
* 15 columns

Issues Identified:

* Missing values in Latitude, Longitude, Category, Footfall, and Revenue
* 20 duplicate records

## Data Cleaning Process

* Replaced missing numerical values using median imputation
* Replaced missing categorical values with "Unknown"
* Removed duplicate records
* Converted date fields to proper datetime format

## Feature Engineering

Created additional features:

* Revenue_Per_Order
* Year
* Month

## Final Output

Cleaned Dataset:

* 180 records
* 18 columns
* No missing values
* No duplicate records

## Project Files

* retail_raw.csv
* cleaned_dataset.csv
* data_cleaning.py
* data_dictionary.xlsx
* final_report.docx

## Conclusion

The dataset was successfully transformed into an analysis-ready format suitable for geospatial analysis, demand forecasting, business intelligence, and retail expansion planning.
